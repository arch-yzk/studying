    コンテナ技術

    とは

    Docker, Kubernetesを始めとする仮想化技術。 従来の問題点である、「イメージが大きい」「起動に時間がかかる」事で取り回しがしづらかった仮想環境を改革。

    コンテナの中は実行するプロセスが依存するルートファイルのみで、ホストOSや他のコンテナをみることすら不可能。 隔離された環境かつ最低限のファイルのみ含まれているので、起動がとても早くセキュア。

    起動

    試しにCentOSを起動してみる。

    arch-yzk >>>> docker run --rm -it centos
Unable to find image 'centos:latest' locally
latest: Pulling from library/centos
3c72a8ed6814: Pull complete
Digest: sha256:76d24f3ba3317fa945743bb3746fbaf3a0b752f10b10376960de01da70685fbd
Status: Downloaded newer image for centos:latest
[root@4967cbf9c2d9 /]#
[root@4967cbf9c2d9 /]#
[root@4967cbf9c2d9 /]# ls
bin  etc   lib    lost+found  mnt  proc  run   srv  tmp  var
dev  home  lib64  media       opt  root  sbin  sys  usr
[root@4967cbf9c2d9 /]# ps a
    PID TTY      STAT   TIME COMMAND
      1 pts/0    Ss     0:00 /bin/bash
     15 pts/0    R+     0:00 ps a
[root@4967cbf9c2d9 /]#

    上記のようにdocker run --rm -it centosコマンドを入力することで、CentOSの最新バージョンのイメージをインストールしつつ実行できる。 このコンテナ内からはホストである私のArchlinuxの情報はみることができず、プロセス自体がコンテナだと認識することもできない。

    コンテナの素晴らしいところは以下である。

    1. OSカーネルがイメージファイルに含まれておらず、普通のプロセスとして起動するため高速
    2. OSカーネルの機能を用いて、通常のプロセスより強力に隔離されている
    3. コンテナ内で依存ファイル群が揃うため、どんなホストでも同様に機能する
